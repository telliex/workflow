# 常见问题

# 1. Mock 引用的 CSS 文件、HTML 和 tpl 文件

在引用HTML时, jest一般都会报错

import tplItem from './item.html'

WARNING

/packages/vip-vconsole/src/log/item.html:1
    ({"Object.<anonymous>":function(module,exports,require,__dirname,__filename,global,jest){<div class="vc-item vc-item-{{logType}} {{if (!noMeta)}}vc-item-nometa{{/if}} {{style}}">

SyntaxError: Unexpected token <

引入 identity-obj-proxy 模块，作用就是更改 import 模块的时候，其导出的是一个无论 key 为何值 value 都等于 key 的一个对象

npm install --save-dev identity-obj-proxy

// jest.config.js
moduleNameMapper: {
    "\\.(html)$": "identity-obj-proxy"
}

# 2. alias 查找路径重命名

一般习惯在webpack.config.js里配置

{
    ...
    resolve: {
        alias: {
            "@": path.join(__dirname, 'src', 'js')
        }
    }
}

当

import xxx from '@/pages/abs'

jest 找不到 '@/pages/abs'，需要在 jest.config.js 添加设定

// jest.config.js
{
    ...
    "jest": {
        "^@/(.*)$": "<rootDir>/src/js/$1"
    }
}

# 3. .mock.calls 缓存之前单元测试的记录

最好都在单元测试中顶部增加

beforeEach(() => jest.resetModules());

这会在运行每次单元测试前重置模块，防止某个单元测试修改了模块, 而影响到了其他单元测试

# 4. mock 文件和 css module 的问题

js文件中引用了css或者本地其他文件，那么就可能测试失败。

在 moduleNameMapper 加入以下設定

"jest": {
    "moduleNameMapper": {
     "\\.(jpg|jpeg|png|gif|eot|otf|webp|svg|ttf|woff|woff2|mp4|webm|wav|mp3|m4a|aac|oga)$": "<rootDir>/test/config/fileMock.js",
     "\\.(css|less)$": "identity-obj-proxy"
 }

// fileMock.js 内容
module.exports = 'test-file-stub';

npm install --save-dev identity-obj-proxy
